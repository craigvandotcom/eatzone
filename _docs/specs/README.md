# Specs Directory

This directory contains all project specifications, guidelines, and standards organized by topic. Each file covers a single topic with descriptive 3-word-or-less filenames.

## File Index

### Design & UI

- **`design-system.md`** - Design system usage patterns, component guidelines, mobile touch interactions
- **`design-tokens.md`** - Color, typography, spacing, and other design token standards
- **`brand-identity.md`** - Brand identity, mission, voice, and visual identity guidelines

### Food Classification

- **`food-zoning.md`** - Comprehensive food zoning algorithm (Green/Yellow/Red zones)
- **`body-compass.md`** - Body Compass reference guide for food tracking and symptom patterns

### Development

- **`development-workflow.md`** - Development scripts, VS Code setup, common tasks
- **`environment-setup.md`** - Environment variables configuration and setup
- **`testing-strategy.md`** - Testing architecture overview (Jest + Playwright)
- **`testing-practices.md`** - Testing best practices, patterns, and project-specific improvements

### Infrastructure

- **`rate-limiting.md`** - Rate limiting specification for API endpoints
- **`supabase-auth.md`** - Supabase authentication guide and demo mode

### Standards

- **`coding-standards.md`** - Comprehensive coding best practices (HTML, CSS, TypeScript, React) and project-specific engineering standards

## When to Reference These Files

**Design & UI work:**

- Building new components → `design-system.md`, `design-tokens.md`
- Branding decisions → `brand-identity.md`

**Food classification:**

- Implementing zoning logic → `food-zoning.md`
- User guidance on food tracking → `body-compass.md`

**Development tasks:**

- Setting up environment → `environment-setup.md`
- Understanding workflow → `development-workflow.md`
- Writing tests → `testing-strategy.md`, `testing-practices.md`

**Infrastructure:**

- API rate limiting → `rate-limiting.md`
- Authentication flows → `supabase-auth.md`

**Code quality:**

- General best practices → `coding-standards.md`
- TypeScript patterns → `coding-standards.md`
- React patterns → `coding-standards.md`
